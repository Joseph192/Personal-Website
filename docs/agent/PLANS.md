# Planning Guide

Use a plan before large changes.

A plan is required for:
- Redesigning the homepage.
- Changing the site structure.
- Adding a backend.
- Adding a CMS.
- Changing routing.
- Adding authentication.
- Large refactors.

## Plan Template

```md
# Plan: [Title]

## Goal

[What we are trying to accomplish.]

## Current Structure

[What exists now.]

## Proposed Changes

[What should change.]

## Files Likely To Change

- `[path]` - [reason]

## Steps

1. [Step one]
2. [Step two]
3. [Step three]

## Verification

- [Build/check command]
- [Manual thing to verify]

## Risks

- [Risk]
- [How to reduce risk]
```

## Rule

For small content updates, do not create a plan. Just make the update cleanly.
