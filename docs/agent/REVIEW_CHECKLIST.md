# Review Checklist

Before finishing, check:

## Correctness

- Does the change solve the requested problem?
- Are edge cases handled?
- Is existing behavior preserved where expected?

## Scope

- Are unrelated refactors avoided?
- Are file changes limited to the task?
- Are generated files or lockfiles changed intentionally?

## Security

- Are inputs validated?
- Are permissions preserved?
- Are secrets and sensitive data protected?

## Tests

- Were relevant tests added or updated?
- Were relevant commands run?
- Are skipped checks explained?

## Maintainability

- Is the code easy to read?
- Does it follow nearby patterns?
- Are comments useful and not redundant?

## Final Response Format

Include:
1. Summary of changes.
2. Tests/checks run.
3. Risks or follow-ups.
